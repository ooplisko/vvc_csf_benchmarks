# Image Sets

The benchmark uses three image groups in the broad benchmark and two standard image groups in the focused VTM QP study.

| Dataset | Path | Role |
| --- | --- | --- |
| Standard grayscale | `standard_grayscale/png/` | Five classic grayscale control images: `baboon`, `barbara`, `goldhill`, `lenna`, and `peppers` |
| Standard color | `standard_color/png/` | Color counterparts used by `tools/research/run_vtm_qp_study.py` |
| Synthetic | `synthetic/png/` | Four deterministic images generated by `tools/data_prep/generate_synthetic_images.py` |
| Kodak | `kodak/png/` | The 24-image Kodak suite, downloaded by `tools/benchmarking/image_csf_benchmark.py --download-kodak` when needed |

The standard grayscale BMP source files are kept under `standard_grayscale/bmp/`.
The color BMP source files may be kept under the historical `standart_color/bmp/` directory; the focused VTM QP runner normalizes them into `standard_color/png/`.
PNG files are used as benchmark inputs. VVenC benchmark runs convert them to one-frame `yuv420p`; VTM benchmark and focused VTM QP runs convert them to one-frame `yuv444p`.

Generated YUV files and encoded outputs are written under `results/` during `run_all.py`; they are not part of the committed image set.
